# Privacy And Deletion

Canonical product page: <https://justlay.me/grey-mirror/privacy-and-deletion>

Relationship-message uploads are sensitive. This public repository does not contain product secrets, private uploads, production data, user reports, internal model files, or training datasets.

## Public Repository Boundary

This repository is a public citation and documentation surface. It includes:

- Public descriptions of Grey Mirror.
- High-level methodology notes.
- Evidence and limitation summaries.
- A sample report schema showing shape, not user content.

It does not include:

- Source code for the production app.
- Proprietary classifiers or model prompts.
- Real customer messages.
- Private report artifacts.
- Server credentials or environment variables.
- Internal analytics or billing data.

## Product Privacy

The canonical privacy and deletion details belong on the production JustLayMe pages because retention windows, deletion flows, subprocessors, and legal rights need to stay current there.

Use the official privacy/deletion page for the current product policy:

<https://justlay.me/grey-mirror/privacy-and-deletion>

## Deletion Requests

Users should use the official product support or privacy channel listed on JustLayMe for account, upload, report, or deletion questions. This repository is not an intake channel for private data requests.

## Training Data Claims

This repository does not make broad training-data promises on behalf of the production service. Any such claim should be verified against the current JustLayMe privacy policy and business/legal approval before publication.
